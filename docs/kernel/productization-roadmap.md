# Productization Roadmap

This roadmap starts after Reality Alignment acceptance is in place. It is intentionally staged so later product work cannot outrun source/docs/script proof.

| Phase | Goal | Current status | Acceptance authority |
| --- | --- | --- | --- |
| 1. Backend Service Facade | Split stable backend service boundaries without changing behavior | implemented | `backend:service-smoke`, `backend:contract-smoke` |
| 2. ToolRuntime V2 | Promote tool execution, typed events, and result storage into a core runtime | implemented | `tool-runtime:smoke`, `patch-verifier:smoke` |
| 3. Code Agent Harness | Build first-class code editing, patching, test, and review loops | implemented | `code-agent:harness-smoke`, `code-agent:fixture-smoke` |
| 4. Benchmark Harness | Add deterministic offline evaluation for code, loop, gateway, GUI, model, and replay cases | implemented | `benchmark:smoke`, `benchmark:*` category smokes |
| 5. Context/Memory V2 | Move compaction, memory, and token accounting into audited core views | implemented | `context:runtime-smoke`, `memory:smoke`, `compaction:runtime-smoke`, `context:budget-smoke` |
| 6. Worktree/Sandbox | Add safe workspace isolation and rollback policies | implemented baseline | `worktree:smoke`, `sandbox:policy-smoke`, `permission:profile-smoke`, `path-policy:smoke`, `command-policy:smoke` |
| 7. Loop Engineering V3 | Add richer loop specs, verifier graphs, and manual gates | pending | Loop recovery and long-run acceptance |
| 8. Dingxu GUI Benchmark | Measure GUI action reliability, speed, focus, and recovery | pending | Real GUI benchmark ledger |
| 9. Gateway real daemon | Package long-running gateway process and restart behavior | pending | Gateway daemon 24h evidence |
| 10. Model production probes | Add live provider health, latency, cost, and capability probes | pending | Provider probe report with redacted secrets |
| 11. Replay golden traces | Maintain stable traces for regressions and incident training | pending | Golden replay diff smoke |
| 12. Mission Control backend APIs | Expose operator state for a mature Web UI | pending | API contract smoke before UI work |
| 13. Security/License | Audit secrets, approvals, dependency licenses, and copied-code provenance | pending | Security/license report |
| 14. CI/Release | Run acceptance gates in clean automation | pending | CI status and generated acceptance report |
| 15. 72h Chaos | Exercise crashes, restarts, stale locks, gateway wakeups, and GUI stuck states | pending | 72h chaos report |
| 16. Cloud worker foundation | Prepare optional cloud coordination without replacing local-first runtime | pending | Cloud worker boundary smoke |

The integrated productization baseline is `npm run productization:phase-smoke`; full release evidence is generated by `npm run acceptance:full`.
