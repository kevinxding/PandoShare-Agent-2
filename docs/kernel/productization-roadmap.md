# Productization Roadmap

This roadmap starts after Reality Alignment acceptance is in place. It is intentionally staged so later product work cannot outrun source/docs/script proof.

| Phase | Goal | Current status | Acceptance authority |
| --- | --- | --- | --- |
| 1. Backend Service Facade | Split stable backend service boundaries without changing behavior | implemented | `backend:service-smoke`, `backend:contract-smoke` |
| 2. ToolRuntime V2 | Promote tool execution, typed events, and result storage into a core runtime | implemented | `tool-runtime:smoke`, `patch-verifier:smoke` |
| 3. Code Agent Harness | Build first-class code editing, patching, test, and review loops | implemented | `code-agent:harness-smoke`, `code-agent:fixture-smoke` |
| 4. Benchmark Harness | Add deterministic offline evaluation for code, loop, gateway, GUI, model, and replay cases | implemented | `benchmark:smoke`, `benchmark:*` category smokes |
| 5. Context/Memory V2 | Move compaction, memory, and token accounting into audited core views | implemented | `context:runtime-smoke`, `memory:smoke`, `compaction:runtime-smoke`, `context:budget-smoke` |
| 6. Worktree/Sandbox | Add safe workspace isolation and rollback policies | implemented baseline | `worktree:smoke`, `sandbox:policy-smoke`, `permission:profile-smoke`, `path-policy:smoke`, `command-policy:smoke` |
| 7. Loop Engineering V3 | Add richer loop specs, verifier graphs, subagent plans, connector plans, skill candidates, and manual gates | implemented baseline | `loop:v3-smoke`, `loop:verifier-graph-smoke`, `loop:skill-candidate-smoke` |
| 8. Dingxu GUI Benchmark | Measure GUI action reliability, speed, focus, approval, and stuck recovery with mock-first and gated real probes | implemented baseline | `gui:benchmark-smoke`, `gui:real-probe-smoke`, `gui:recovery-benchmark-smoke` |
| 9. Gateway real daemon | Package foreground daemon process, heartbeat, watchdog, local webhook, and bounded gateway service ticks | implemented baseline | `daemon:foreground-smoke`, `gateway:service-smoke`, `gateway:webhook-smoke`, `gateway:watchdog-smoke` |
| 10. Model production probes | Add offline/static/mock provider health, capability, budget, fallback, and secret-safe reports | implemented baseline | `model:probe-smoke`, `model:probe-offline-smoke`, `model:probe-report-smoke`, `model:probe-fallback-smoke` |
| 11. Replay golden traces | Maintain deterministic traces for replay regression defense | implemented baseline | `replay:golden-smoke`, `replay:golden-diff-smoke`, `replay:golden-report-smoke`, `replay:golden-update-dry-run` |
| 12. Mission Control backend APIs | Expose operator state for a mature Web UI without implementing UI | implemented baseline | `mission:api-smoke`, `mission:contract-smoke`, `mission:action-smoke` |
| 13. Security/License | Audit secrets, approvals, dependency licenses, and copied-code provenance | implemented baseline with blocker | `security:secret-smoke`, `security:license-smoke`, `security:boundary-smoke`, `security:report-smoke` |
| 14. CI/Release | Run acceptance gates in clean automation and package dry-runs | implemented baseline | `release:smoke` |
| 15. 72h Chaos | Exercise crashes, restarts, stale locks, gateway wakeups, and GUI stuck states | implemented baseline | `chaos:smoke`, `chaos:report-smoke`, `chaos:72h-plan` |
| 16. Cloud worker foundation | Prepare optional cloud coordination without replacing local-first runtime | implemented baseline | `cloud:worker-smoke`, `cloud:job-envelope-smoke`, `cloud:artifact-manifest-smoke`, `cloud:coordinator-smoke` |

The integrated Wave 1 baseline is `npm run productization:phase-smoke`; the integrated Wave 2 baseline is `npm run productization:wave-2-smoke`; final backend productization baseline is `npm run productization:final-smoke`; full release evidence is generated by `npm run acceptance:full`.
